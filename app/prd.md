단모음 조합 기반 한글 키보드 앱 PRD (Product Requirements Document)1. 개요 (Overview)본 문서는 '단모음 조합 기반'의 새로운 안드로이드 한글 키보드 앱 개발을 위한 요구사항을 정의합니다. 기존의 천지인, 나랏글 방식과 달리, 사용자가 자음과 단모음을 직접 눌러 조합하는 직관적인 방식을 제공함으로써, 스마트폰 환경에서 한글 입력의 새로운 표준을 제시하는 것을 목표로 합니다. 이 키보드는 단순성, 빠른 학습 곡선, 그리고 정확성에 중점을 둡니다.2. 목표 (Goals)•핵심 목표: 사용자가 최소한의 학습으로 빠르고 정확하게 한글을 입력할 수 있는 키보드를 제공한다.•사용자 경험: 직관적인 키 조합 로직을 통해 오타를 줄이고, 리듬감 있는 타이핑 경험을 제공한다.•기술 목표: 경량화된 엔진으로 어떤 안드로이드 기기에서도 부드럽고 빠른 반응성을 보장한다.3. 기능 요구사항 (Functional Requirements)FR-1: 키보드 레이아웃•기본 구성: 자음과 단모음 키들이 명확히 분리되어 배치되어야 한다.◦자음 영역: ㄱ, ㄴ, ㄷ, ㄹ, ㅁ, ㅂ, ㅅ, ㅇ, ㅈ, ㅊ, ㅋ, ㅌ, ㅍ, ㅎ◦단모음 영역: ㅏ, ㅑ, ㅓ, ㅕ, ㅗ, ㅛ, ㅜ, ㅠ, ㅡ, ㅣ◦기타 키: Shift, Backspace, Space, Enter, 숫자/기호 전환•된소리(ㄲ, ㄸ, ㅃ, ㅆ, ㅉ) 입력:◦Shift 키를 누른 상태에서 해당 자음(ㄱ, ㄷ, ㅂ, ㅅ, ㅈ)을 누르면 된소리가 입력된다.◦(대안) 해당 자음 키를 길게 누르거나(Long Press), 두 번 빠르게 탭(Double Tap)하여 입력하는 방식을 옵션으로 제공한다.•레이아웃 배치: 사용자의 양손 파지 시 동선이 가장 효율적인 배치를 연구하여 적용한다. (예: 왼손-자음, 오른손-모음)FR-2: 한글 조합 로직 (IME Engine)•초성-중성-종성 조합: 표준 한글 조합 규칙을 따른다.◦자음 입력 -> 초성◦+ 모음 입력 -> 중성 (초성과 결합)◦+ 자음 입력 -> 종성 (앞 글자와 결합)•이중 모음 조합: 사용자가 단모음을 순서대로 입력하면 자동으로 이중 모음으로 조합되어야 한다.◦ㅗ + ㅏ → ㅘ◦ㅜ + ㅓ → ㅝ◦ㅜ + ㅣ → ㅟ◦ㅡ + ㅣ → ㅢ◦ㅐ + ㅣ → ㅒ (X) -> ㅑ+ㅣ -> ㅒ 또는 ㅏ+ㅣ->ㅐ 와 같은 규칙 재정의 필요◦규칙 예시:▪ㅗ 입력 상태에서 ㅏ,ㅐ,ㅣ 입력 시 각각 ㅘ, ㅙ, ㅚ로 조합▪ㅜ 입력 상태에서 ㅓ,ㅔ,ㅣ 입력 시 각각 ㅝ, ㅞ, ㅟ로 조합▪ㅡ 입력 상태에서 ㅣ 입력 시 ㅢ로 조합•종성 분리/재조합:◦값 입력 상태에서 모음 ㅏ를 입력하면 갑 + 사로 자동 분리 및 재조합되어야 한다. (갑사)◦안 입력 상태에서 ㅈ을 입력하면 앉으로 겹받침이 형성되어야 한다.FR-3: 부가 기능•영문/숫자/기호 키보드:◦한/영 전환 키를 통해 표준 QWERTY 영문 키보드로 전환된다.◦?123 전환 키를 통해 숫자 및 기본 기호 키패드로 전환된다.◦기호 키패드에서 =\< 와 같은 추가 키를 통해 2차 기호 세트로 전환된다.•키 입력 피드백:◦키를 누를 때 시각적 피드백 (e.g., 키 팝업)을 제공한다.◦설정에서 소리 및 진동 피드백을 켜고 끌 수 있다.•기본 편집 기능:◦Backspace: 한 글자(자소 단위)씩 지운다. 길게 누르면 단어 단위로 빠르게 삭제한다.◦Space: 공백을 입력한다.◦Enter: 줄 바꿈을 실행한다.4. 비기능 요구사항 (Non-Functional Requirements)•성능 (Performance): 키 입력 시 50ms 이내의 반응 속도를 유지하여 사용자가 딜레이를 느끼지 않도록 한다.•호환성 (Compatibility): Android 8.0 (Oreo) 이상 버전과 호환되어야 한다.•UI/UX: 미니멀하고 깔끔한 디자인을 적용한다. 키보드 높이 조절 기능을 제공하여 사용자 편의를 높인다.5. 향후 개선 계획 (Future Roadmap)•v1.1: 다크 모드/라이트 모드 테마 지원, 사용자 정의 테마 기능 추가•v1.2: 사용자 사전 기능, 자동 완성 및 오타 교정 기능 추가•v1.5: 클립보드 히스토리, 한 손 모드 기능 추가•v2.0: 스티커, 이모지, GIF 검색 기능 연동


단모음 조합 기반 한글 키보드 앱 PRD (v1.1 - 레이아웃 반영)1. 개요 (Overview)본 문서는 '15키 기반의 단모음 조합' 한글 키보드 앱 개발 요구사항을 정의합니다. 사용자는 이미지에 명시된 3x5 배열의 기본 자음과 단모음 키를 조합하여 모든 한글을 입력할 수 있습니다. 이 방식은 최소한의 키로 직관적인 한글 입력을 가능하게 하는 것을 목표로 합니다.2. 목표 (Goals)•핵심 목표: 제한된 개수의 키 조합을 통해, 스마트폰 환경에 최적화된 빠르고 정확한 한글 입력을 제공한다.•사용자 경험: 키의 위치를 빠르게 암기하고, 리듬감 있는 조합을 통해 타이핑의 즐거움을 제공한다.•기술 목표: 경량화된 조합 엔진으로 저사양 기기에서도 부드러운 반응성을 보장한다.3. 기능 요구사항 (Functional Requirements)FR-1: 키보드 레이아웃 (수정)•한글 키보드 (15키 배열):◦핵심 키 영역 (3x5 그리드):▪1행: ㄱ ㄴ ㄷ | ㅏ ㅓ▪2행: ㄹ ㅁ ㅂ | ㅗ ㅜ▪3행: ㅅ ㅇ ㅈ | ㅡ ㅣ◦기능 키 영역 (하단 행):▪🔄 (Shift/특수기능): 된소리 입력 및 특수문자 전환에 사용된다.▪🌐 (입력기 전환): 한/영 및 다른 시스템 키보드로 전환한다. (이미지의 지구본 아이콘)▪[ ] (스페이스 바): 공백을 입력한다.▪↵ (Enter): 줄 바꿈을 실행한다.▪← (Backspace): 글자를 삭제한다.•영문 키보드:◦🌐 키를 통해 표준 QWERTY 배열의 영문 키보드로 전환된다. (이미지 참조)FR-2: 한글 조합 로직 (IME Engine) (수정)•기본 조합: 초성-중성-종성 순서의 표준 한글 조합 규칙을 따른다.•된소리 (ㄲ, ㄸ, ㅃ, ㅆ, ㅉ) 입력:◦🔄 (Shift) 키를 누른 상태에서 해당 자음(ㄱ, ㄷ, ㅂ, ㅅ, ㅈ)을 누르면 된소리가 입력된다.◦예시: 🔄 + ㄱ → ㄲ•거센소리 (ㅋ, ㅌ, ㅍ, ㅊ, ㅎ) 입력:◦기본 자음을 빠르게 두 번 탭(Double Tap)하여 입력한다.◦규칙:▪ㄱ → ㅋ▪ㄷ → ㅌ▪ㅂ → ㅍ▪ㅈ → ㅊ▪ㅇ → ㅎ•복모음 및 이중모음 조합:◦단모음을 순서대로 입력하면 자동으로 복모음/이중모음으로 조합된다.◦규칙 예시:▪'ㅣ' 계열 조합: ㅣ + ㅏ → ㅑ, ㅣ + ㅓ → ㅕ, ㅣ + ㅗ → ㅛ, ㅣ + ㅜ → ㅠ▪'ㅣ' 결합 조합: ㅏ + ㅣ → ㅐ, ㅓ + ㅣ → ㅔ▪'ㅗ/ㅜ' 계열 조합: ㅗ + ㅏ → ㅘ, ㅗ + ㅐ → ㅙ, ㅜ + ㅓ → ㅝ, ㅜ + ㅔ → ㅞ▪기타 조합: ㅡ + ㅣ → ㅢ, ㅗ + ㅣ → ㅚ, ㅜ + ㅣ → ㅟ•종성 분리/재조합: (기존과 동일)◦값 입력 상태에서 모음 ㅏ를 입력하면 갑 + 사로 자동 분리 및 재조합되어야 한다. (갑사)FR-3: 부가 기능 (수정)•입력기 전환:◦🌐 키를 눌러 한글 15키 ↔ 영문 QWERTY 간에 전환한다.•키 입력 피드백: (기존과 동일)◦키를 누를 때 시각적 피드백(키 팝업)을 제공한다.◦설정에서 소리 및 진동 피드백을 켜고 끌 수 있다.•기본 편집 기능: (기존과 동일)◦← (Backspace): 한 글자(자소 단위)씩 지운다. 길게 누르면 단어 단위로 빠르게 삭제한다.◦[ ] (Space): 공백을 입력한다.◦↵ (Enter): 줄 바꿈을 실행한다.(4, 5번 항목은 기존 PRD와 동일하게 유지)
